Data Model

SQLite schema — what gets indexed and how

SQLite Schema
┌─────────────────────────────────┐
│           sessions              │
├─────────────────────────────────┤
│ id          TEXT PK             │  ── session UUID
│ tool        TEXT                │  ── "claude" | "copilot" | "codex"
│ project     TEXT                │  ── project path / name
│ cwd         TEXT                │  ── working directory
│ git_branch  TEXT                │  ── branch at time of session
│ started_at  INTEGER             │  ── epoch ms
│ ended_at    INTEGER             │  ── epoch ms (last message)
│ message_count INTEGER           │  ── total messages
│ source_file TEXT                │  ── path to original JSONL
│ file_mtime  INTEGER             │  ── for change detection
│ file_size   INTEGER             │  ── for change detection
└──────────┬──────────────────────┘
           │ 1:N
           ▼
┌─────────────────────────────────┐
│           messages              │
├─────────────────────────────────┤
│ id          INTEGER PK          │
│ session_id  TEXT FK             │  ── references sessions.id
│ uuid        TEXT                │  ── message UUID
│ parent_uuid TEXT                │  ── for conversation threading
│ role        TEXT                │  ── "user" | "assistant" | "system"
│ content     TEXT                │  ── message text (stripped)
│ type        TEXT                │  ── "text" | "tool_use" | "tool_result"
│ timestamp   INTEGER             │  ── epoch ms
│ token_estimate INTEGER          │  ── rough token count
└─────────────────────────────────┘

┌─────────────────────────────────┐
│       messages_fts (FTS5)       │
├─────────────────────────────────┤
│ content     TEXT                │  ── full-text indexed
│ role        TEXT                │  ── filterable
│ session_id  TEXT                │  ── for join back
└─────────────────────────────────┘

┌─────────────────────────────────┐
│        tool_uses                │
├─────────────────────────────────┤
│ id          INTEGER PK          │
│ message_id  INTEGER FK          │
│ session_id  TEXT FK             │
│ tool_name   TEXT                │  ── "Read", "Edit", "Bash", etc.
│ file_path   TEXT                │  ── target file if applicable
│ timestamp   INTEGER             │
└─────────────────────────────────┘

Key Points

  • FTS5 virtual table on message content — enables instant full-text search with ranking
  • tool_uses table — enables analytics like "most edited files", "tool usage breakdown"
  • file_mtime + file_size on sessions — chokidar detects changes, only re-parses if these differ (incremental indexing)
  • token_estimate — derived from content length (rough heuristic: chars / 4), powers token usage analytics
  • Parsers per tool: Each CLI has its own parser (Claude JSONL, Copilot JSON, Codex state JSON) that normalizes into this unified schema
